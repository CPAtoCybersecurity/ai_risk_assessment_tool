# Conversation Starters / Starter Prompts

> Use the same four starter prompts on every platform that supports them (Microsoft 365 Copilot Agent Builder, Custom GPT, and Gemini Gem all surface starter prompts on the agent's launch screen). Claude Project doesn't have a native starter-prompt field — paste these into the project description or use them as a quick-start cheatsheet.

## The Four Starters

1. **Create a risk assessment that is ready for CISO review.**
2. **I'm assessing a new AI system before production approval — start the intake.**
3. **Help me evaluate a third-party AI tool we're considering buying.**
4. **Recommend security controls for an AI feature I'm scoping.**

## Why these four

These four cover the moments a GRC professional is most likely to reach for the agent:

- **#1 (CISO review)** is the *deliverable-first* path: the analyst has done the work and needs a polished, executive-ready document with overall risk rating, top findings, and a clear recommendation.
- **#2 (production approval)** is the *front-door* path — kicks off the full INTAKE → trifecta → tier → threats → checklist → rating → output flow for an internally-built AI system.
- **#3 (vendor evaluation)** covers third-party AI procurement — a major real-world GRC scenario where the analyst is reviewing someone else's system before signing a contract.
- **#4 (control recommendations)** is the *shift-left* path — the analyst engages early, before code is written, to scope controls into the design rather than retrofit them after the fact.

The agent applies the Lethal Trifecta and the three-tier model on every assessment automatically; users don't need to invoke those concepts by name.

## Adapting

If you want a fifth or sixth on platforms that allow more than four (none currently do), good additions:

- *Identify what could go wrong with this AI feature.* (threat-modeling in plain language)
- *Compare two AI architecture options and tell me which carries more risk.*
- *Map this AI system's controls to NIST CSF 2.0.*
- *Find the highest-risk gaps in this AI system's design.*

The order of #1–#4 signals the recommended workflow paths to the user — keep #1 (CISO-review deliverable) first as the most common high-value request, with intake/vendor/scoping following.
